# 🎯 **Assignment 3(a) Explanation (Ready to Speak)**

## 🧾 **1. Introduction**

This program implements a **lexical analyzer using LEX (Flex)** for identifying **parts of speech in a subset of the English language** .

A lexical analyzer is the **first phase of a compiler**, which reads input and breaks it into meaningful units called **tokens**.

In this program, tokens are:

* Noun
* Pronoun
* Verb
* Adjective
* Adverb
* Preposition
* Conjunction
* Interjection

---

## 🎯 **2. Objective**

The objective is to:

* Read an English sentence as input
* Identify each word’s **part of speech**
* Print the category
* **Without using a symbol table**

---

## ⚙️ **3. Working of the Program**

This program is written using **LEX**, which works on **pattern matching**.

The code has 3 sections:

---

### 🔹 (1) Definition Section

```c
%{
#include <stdio.h>
%}
```

👉 Used to include header files

---

### 🔹 (2) Rules Section (Main Logic)

This is the most important part.

Example:

```lex
"he"|"she"|"we" { printf("Pronoun: %s\n", yytext); }
```

👉 Meaning:

* If input matches `"he"`, `"she"`, etc.
* It prints → Pronoun

---

Another example:

```lex
[a-zA-Z]+ { printf("Unknown: %s\n", yytext); }
```

👉 If word is not defined → it is treated as **Unknown**

---

### 🔹 (3) User Section

```c
int main() {
    yylex();
}
```

👉 `yylex()`:

* Reads input
* Matches patterns
* Executes corresponding actions

---

## 🧠 **4. Important Concepts**

### 🔸 What is `yytext`?

* It stores the **matched word**

Example:

```c
Input: he
yytext = "he"
```

---

### 🔸 What is Pattern Matching?

* Matching input with predefined rules

---

### 🔸 Why order matters?

* LEX checks rules **top to bottom**
* First match is executed

---

## 💻 **5. Example Execution**

### Input:

```id="r4dx2q"
he walk quickly and she read
```

### Output:

```id="kl3c2o"
Pronoun: he
Verb: walk
Adverb: quickly
Conjunction: and
Pronoun: she
Verb: read
```

---

## ⚠️ **6. Limitations**

* Only predefined words are recognized
* Words like **“walking”, “runs”** may show as Unknown
* No grammar checking
* No symbol table (no storage)

---

## 🔍 **7. Why Symbol Table is NOT used here**

In this assignment:

* We only **identify tokens**
* We do NOT store them

Symbol table is used in later compiler phases, not required here

---

## 🧾 **8. Conclusion (Say confidently)**

This program successfully demonstrates a lexical analyzer using LEX for identifying parts of speech in a subset of English language using pattern matching, without using a symbol table.
