# 🎯 **Assignment 3(c) Explanation (Ready to Speak)**

## 🧾 **1. Introduction**

This program implements a **lexical analyzer using LEX (Flex)** for a **subset of the C programming language without using a symbol table** .

A lexical analyzer is the **first phase of a compiler**, which scans the input program and converts it into **tokens**.

In this program, the tokens include:

* Keywords
* Identifiers
* Numbers
* Operators
* Punctuation symbols
* Strings

---

## 🎯 **2. Objective**

The objective is to:

* Read C language input
* Identify different types of tokens
* Print their category
* **Without storing them in a symbol table**

---

## ⚙️ **3. Working of the Program**

This program is implemented using **LEX**, which works on **pattern matching rules**.

It has 3 main sections:

---
